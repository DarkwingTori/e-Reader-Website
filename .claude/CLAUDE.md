## What This Project Is

A fullstack web platform that serves as the definitive modern resource for Nintendo e-Reader cards (GBA, 2001–2008). It combines:
- A complete searchable card database (every card ever released, US/JP/AU)
- A user collection tracker
- An interactive e-card replication tool that generates functional dot code cards 

See [`docs/PROJECT_SPEC.md`](docs/PROJECT_SPEC.md) for the full vision.

## Core Rules for This Project

### General
- Always use TypeScript — no plain `.js` files
- Prefer explicit types over `any`
- Use path aliases (`@/components`, `@/lib`, `@/types`) — configured in `tsconfig.json` and `vite.config.ts`
- All environment variables go in `.env.local` and must be typed in `src/types/env.d.ts`

### Frontend
- Framework: Vite + React + TypeScript
- Styling: CSS Modules scoped per component, with global CSS custom properties for theming
- Per-series theming is applied via `data-series="..."` on wrapper elements — never hardcode series colors inline
- All design decisions originate in Figma first — do not invent spacing, colors, or typography not in the design system
- Figma tokens are exported via Tokens Studio → `src/tokens/tokens.json` → compiled by Style Dictionary into `src/tokens/generated/`
- Never import raw hex values directly in components — always use CSS variables from the token system

### Components
- One component per file
- Co-locate styles: `CardGrid.tsx` + `CardGrid.module.css` in the same folder
- Co-locate tests: `CardGrid.test.tsx` in the same folder
- Shared UI components live in `src/components/ui/`
- Feature components live in `src/features/<feature-name>/`

### Backend / Supabase
- All database access goes through typed Supabase client in `src/lib/supabase.ts`
- Use Supabase generated types from `src/types/supabase.ts` — regenerate with `npm run types:supabase` after schema changes
- Row Level Security (RLS) must be enabled on all tables — never disable it
- Storage bucket access follows the structure in `docs/DATABASE_SCHEMA.md`
- Edge Functions live in `supabase/functions/` and are written in TypeScript (Deno)

### e-Card Replication Pipeline (Last thing to be worked on)
- The build pipeline (RGBDS + NEDC + Python scripts) runs server-side only — never expose toolchain logic to the client
- All card generation requests go through the Supabase Edge Function at `supabase/functions/generate-card/`
- Generated files are stored in Supabase Storage under `/generated/`
- See `docs/REPLICATION_PIPELINE.md` for full pipeline details before touching anything in `supabase/functions/generate-card/`
- Will download repositories in GitHub to share more available information
